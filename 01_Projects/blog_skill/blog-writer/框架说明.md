## 🎯 框架设计理念

### 与现有 Skill 的对比

| 特性 | Blog Writer | X Writer | XHS Writer |
|------|------------|----------|------------|
| **字数** | 1500-3000 | 50-280 | 800-1500 |
| **工作流** | 采访 → 结构推荐 → 大纲确认 → 分段生成 | 采访 → 公式选择 → 3版本并行 | 采访 → 风格推荐 → 生成+标题 |
| **输出** | 1篇完整文章 + 5标题 + Meta | 3个版本（不同长度） | 1篇 + 3标题 |
| **特点** | 深度论证、结构化 | 快速冲击、并行输出 | 情绪共鸣、爆款公式 |
| **确认机制** | 分段确认（大纲 → 每段） | 一次性输出3版本 | 风格确认后生成 |

### 核心创新点

1. **分段生成机制**
   - 问题：长文一次性生成容易跑偏
   - 解决：大纲确认 → 分段生成 → 逐段确认 → 继续
   - 好处：质量可控、方向不跑偏、用户参与度高

2. **类型映射系统**
   - 问题：博客类型多样，通用模板不适用
   - 解决：4 种类型（观点论证/案例分析/实用指南/深度思考）+ 对应样本
   - 好处：风格精准、结构清晰

3. **素材追踪机制**
   - 问题：采访获得的素材容易在生成时遗漏
   - 解决：大纲阶段生成"素材清单"，生成时强制使用
   - 好处：确保独特观点都被融入文章

4. **多层次输出**
   - 不只生成正文，还包含：
     - 5 个不同风格的标题选项
     - SEO Meta Description
     - 标签建议
     - 图片建议
     - 相关阅读推荐

## 🔧 核心文件解析

### 1. SKILL.md（AI 指令文件）

**核心章节**:
- **工作流指令**：6 个步骤（环境检测 → 采访 → 结构推荐 → 大纲 → 生成 → 保存）
- **示例工作流**：完整的对话示例
- **输出格式规则**：排版、语气、节奏控制
- **高级功能**：续写、素材搜索、多语言、图片建议
- **关键规则**：必须遵守的黄金法则

**设计亮点**:
```markdown
### Step 3: 大纲生成与确认
在开始写作前，生成详细大纲供用户确认：

**大纲格式**:
[详细的大纲模板，包含每部分的论点、素材、字数]

**询问用户**: "这个大纲 OK 吗？需要调整哪部分？确认后我开始写作。"
```

→ 这确保了方向正确后才开始写作，避免浪费。

### 2. README.md（用户手册）

**核心章节**:
- **快速开始**：如何激活 skill
- **工作流程**：6 步详解 + 可视化表格
- **示例会话**：完整的用户-AI 对话
- **输出特点**：风格、结构、排版
- **最佳实践**：DO/DON'T 清单
- **故障排查**：常见问题 Q&A

**设计亮点**:
```markdown
| 结构类型 | 适用场景 | 字数 |
|---------|---------|------|
| **观点论证型** | 推翻常见认知、建立新视角 | 2000-2500字 |
| **案例分析型** | 有具体案例和数据支撑 | 1800-2200字 |
| **实用指南型** | 教程、方法论、工具介绍 | 1500-2000字 |
| **深度思考型** | 趋势分析、哲学思考 | 2500-3500字 |
```

→ 让用户快速理解"什么时候用什么结构"。

### 3. samples/README.md（样本索引）

**核心功能**:
- **类型映射表**：4 种类型 × 样本列表
- **快速选择流程图**：帮助 AI 快速匹配合适样本
- **使用方法**：3 步（识别 → 推荐 → 读取）
- **Token 管理**：确保不会一次性加载所有样本

**设计亮点**:
```markdown
### Step 3: 读取样本（仅在用户选择后）
用户选择某个样本后，读取对应文件：

**注意**:
- 只读取用户选择的那 1 个样本
- 读取后，在生成时模仿其风格特点
- 不要照抄内容，而是学习结构和语气
```

→ 避免 context 爆炸，按需加载。

## 🎨 风格系统

### Felix 的 Blog 语气特征

**DO**（必须有的）:
- ✅ 短句为主（长句不超过 25 字）
- ✅ 口语化但有深度："你可能觉得..."、"但实际上..."
- ✅ 具体化表达：用数字、名字、场景
- ✅ 有立场但不极端："我认为"而非"绝对是"

**DON'T**（必须避免）:
- ❌ 模糊词："可能"、"也许"、"似乎"
- ❌ 空话："在当今时代"、"众所周知"
- ❌ 学术腔调、长从句套从句

### 结构可视化原则

```markdown
**段落**: 每段 2-4 句，不超过 150 字
**小标题**: 使用 ### 或 ####，简洁有力
**强调**: 关键论点用 **加粗**
**列表**: 多用 - 或 1. 提升可读性
**引用**: 重要观点用 > 引用块突出
**图片占位**: 需要配图的地方用 [图: 描述] 标注
```

### 节奏控制

```
开场（快节奏）
  ↓ 用场景/数据吸引
主体（稳定节奏）
  ↓ 每2-3段有节奏变化（问句/列表/案例）
结尾（加速）
  ↓ 给明确行动
```

## 🔄 工作流详解

### Step 0: 环境检测
```
if (user正在编辑某篇blog) {
  询问："继续完善" or "新建文章"？
}
读取核心资源（如未加载）
```

### Step 1: 深度采访
```
必问5题：
1. 话题是什么？
2. 核心论点是什么？
3. 独特素材是什么？
4. 目标读者是谁？
5. 期望读者做什么？

高级追问（按需）：
- 常见误区？
- 踩过的坑？
- 反直觉洞察？
```

### Step 2: 结构推荐
```
读取 samples/README.md
分析话题类型
推荐 2-3 个结构选项
等待用户选择
if (用户选择某个) {
  读取对应样本（只读 1 个）
} else {
  使用通用模板
}
```

### Step 3: 大纲生成
```
生成详细大纲：
- 每部分的论点
- 要用的素材
- 预估字数
- 论证逻辑

生成素材清单（checkbox）
询问用户确认
```

### Step 4: 分段生成
```
生成"开场 + 主体1" → 等待确认
  ↓ OK
生成"主体2" → 等待确认
  ↓ OK
生成"主体3（如有）+ 结尾" → 完成

每段必须：
- 遵循大纲
- 使用 Felix 语气
- 融入素材清单中的内容
```

### Step 5: 标题与元信息
```
生成：
- 5 个标题选项（不同风格）
- Meta Description（120-160 字符）
- Tags 建议（3-5 个）
```

### Step 6: 保存输出
```
保存路径: 01_Projects/blog_sample/YYYY-MM-DD-[topic].md
格式: 完整 frontmatter + 正文 + 相关阅读

输出统计：
- 总字数
- 段落数
- 预计阅读时间
- 下一步建议（配图、校对、内链、SEO）
```

| 资源 | 大小 | 何时加载 |
|------|------|---------|
| SKILL.md | ~4000 tokens | Skill 激活时（自动） |
| Blog写作结构.md | ~2000 tokens | Step 0（必读） |

| 单个样本 | ~2500 tokens | Step 2（用户选择后） |
| **总计** | **~11000 tokens** | **全流程** |

### 按需加载策略

```
Step 0: 核心资源（必读）
  ├── Blog写作结构.md
  └── 个人背景.md

Step 2: 样本系统（按需）
  ├── samples/README.md (索引，先读)
  └── 具体样本 (用户选择后才读，只读1个)

高级功能: 素材库（按需搜索）
  ├── 使用 Grep 搜索关键词
  └── 只读取匹配的部分，不全量加载
```

## 🎯 如何使用此框架

### 激活 Skill
```
/blog-writer
```

或直接说：
```
"我想写一篇关于 [话题] 的博客"
```

### 完整流程（首次使用）
1. **创建核心资源**（如果还没有）：
   - `Blog写作结构.md`
   - `个人背景.md`

2. **补充样本库**（推荐至少 3 个样本）：
   - 选择你已有的高质量博客
   - 按照样本格式添加到 `samples/` 目录
   - 更新 `samples/README.md` 索引

3. **首次写作测试**：
   - 激活 skill
   - 跟随 AI 的采访
   - 确认大纲
   - 分段生成并确认
   - 检查输出质量

4. **迭代优化**：
   - 根据实际使用效果调整 SKILL.md 中的指令
   - 补充更多样本
   - 扩充素材库
