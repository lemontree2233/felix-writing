# Blog Writer Skill

Felix 的专属 Blog 创作助手 - 生成结构化、有深度的长文博客，保持独特视角和风格。

## 快速开始

**激活 skill:**
```
/blog-writer
```

**或者直接告诉 Claude:**
```
"我想写一篇关于 [话题] 的博客"
"帮我写一篇 blog 讨论 [主题]"
```

---

## 这个 Skill 能做什么

1. **深度采访** - 通过结构化提问，挖掘你的独特观点和素材
2. **推荐结构** - 基于话题类型，推荐最适合的文章结构
3. **生成大纲** - 提供详细大纲供确认，确保方向正确
4. **分段写作** - 分段生成正文，每段确认后再继续
5. **标题生成** - 提供 5 个不同风格的标题选项
6. **保存输出** - 自动保存到 blog_sample 目录，带完整 frontmatter

## 工作流程

### 1️⃣ 深度采访（必经步骤）
AI 会问你 5 个核心问题：
- 话题是什么？（具体聚焦）
- 核心论点是什么？（一句话）
- 有什么独特素材？（数据/经历/案例）
- 目标读者是谁？（技术/创业/普通用户）
- 期望读者做什么？（行动导向）

**为什么要采访？**
因为好的博客来自具体的观点和素材，不是泛泛而谈。

### 2️⃣ 结构推荐
基于你的回答，AI 会推荐 2-3 种文章结构：

| 结构类型 | 适用场景 | 字数 |
|---------|---------|------|
| **观点论证型** | 推翻常见认知、建立新视角 | 2000-2500字 |
| **案例分析型** | 有具体案例和数据支撑 | 1800-2200字 |
| **实用指南型** | 教程、方法论、工具介绍 | 1500-2000字 |
| **深度思考型** | 趋势分析、哲学思考 | 2500-3500字 |

你可以选择一个，或者跳过使用通用结构。

### 3️⃣ 大纲确认
生成详细大纲，包括：
- 每部分的论点
- 要用的素材
- 预估字数
- 论证逻辑

**确认后才开始写作**，避免方向跑偏。

### 4️⃣ 分段生成
不会一次性生成全文，而是分段确认：
1. 生成"开场 + 主体部分 1" → 等你确认
2. 生成"主体部分 2" → 等你确认
3. 生成"主体部分 3（如有）+ 结尾" → 完成

每段你都可以要求调整，然后再继续。

### 5️⃣ 标题与元信息
正文完成后，AI 会提供：
- **5 个标题选项**（不同风格：直击痛点型、数据型、反常识型等）
- **Meta Description**（SEO 用，120-160字符）
- **标签建议**（主题标签、技术标签、受众标签）

### 6️⃣ 保存输出
自动保存到:
```
01_Projects/blog_sample/YYYY-MM-DD-[topic-slug].md
```

文件包含完整的 frontmatter:
```yaml
---
title: "你选定的标题"
date: 2026-01-27
tags: [tag1, tag2, tag3]
description: "Meta Description"
status: draft
---
```

## 输出特点

### 文章结构
- **字数**: 通常 1500-3000 字（根据话题深度调整）
- **开场**: 场景/数据/问题切入，快速吸引
- **主体**: 2-3 个核心论证部分，每部分有小标题
- **结尾**: 总结 + 行动建议 + 金句/思考题

### 风格特色
**Felix 的 Blog 语气**：
- ✅ 短句为主，长句不超过 25 字
- ✅ 口语化但有深度："你可能觉得..."、"但实际上..."
- ✅ 具体化表达：用数字、名字、场景
- ✅ 有立场但不极端："我认为"而非"绝对是"

**避免的风格**：
- ❌ 模糊词："可能"、"也许"、"似乎"
- ❌ 空话："在当今时代"、"众所周知"
- ❌ 学术腔调、长从句套从句

### 排版优化
- **小标题**: 2-3 层结构，简洁有力
- **强调**: 关键论点用 **加粗**
- **列表**: 多用列表提升可读性
- **引用**: 重要观点用 `>` 引用块突出
- **图片占位**: 需要配图的地方标注 `[图: 描述]`

## 高级功能

### 1. 续写模式
如果你正在编辑一篇已有的博客草稿：
```
"继续写下一部分"
"扩充第三段"
"重写结尾"
```

AI 会先读取现有内容，保持风格一致。

### 2. 素材搜索
如果需要更多素材：
```
"帮我搜索一些关于 AI 游戏开发的案例"
"找一些数据支持这个论点"
```

AI 会在素材库中搜索，或使用 WebSearch 找最新信息。

### 3. 多语言支持
- 接受中英文混合输入
- 默认输出与你的输入语言一致
- 可指定："用英文写"或"用中文写"

### 4. 图片建议
AI 会在适当位置建议配图：
```
[图片建议: 对比图 - 传统开发 vs AI 开发的时间差异]
[配图类型: 流程图]
```

## 最佳实践

### ✅ DO（推荐做法）

1. **准备好素材**
   - 收集好数据、案例、个人经历
   - 明确你的核心观点
   - 知道你想说服读者做什么

2. **积极参与采访**
   - 认真回答 AI 的每个问题
   - 提供具体细节，不要说"随便"
   - 如果想到新素材，随时补充

3. **确认大纲后再写**
   - 大纲是方向盘，确保方向正确
   - 发现问题立即调整大纲
   - 不要等全文写完再推翻重来

4. **分段确认**
   - 每段生成后仔细读一遍
   - 发现问题立即指出
   - 小调整比大返工高效

5. **选标题时考虑 SEO**
   - 包含核心关键词
   - 符合搜索习惯
   - 但也要有吸引力

### ❌ DON'T（避免做法）

1. **不要跳过采访**
   - "随便写点什么"会导致空洞内容
   - 采访是挖掘独特性的关键

2. **不要一次性生成全文**
   - 分段生成是为了质量控制
   - 一次生成很难保证全文质量

3. **不要提供泛泛话题**
   - ❌ "写一篇关于 AI 的文章"
   - ✅ "写一篇关于 AI 如何让 solo founder 可以替代小团队的文章"

4. **不要忽略大纲**
   - 大纲不只是形式
   - 它决定了文章的逻辑和节奏

5. **不要期望一次完美**
   - 好文章是迭代出来的
   - 第一版是基础，后续可以继续优化

## 故障排查

**Q: AI 生成的内容太空泛，没有深度？**
A: 回到采访阶段，提供更具体的素材（数据、案例、个人经历）。空泛通常是因为素材不足。

**Q: 风格不对，太学术或太随意？**
A: 明确告诉 AI："更口语化一点"或"更专业一点"。也可以提供一篇你喜欢的文章作为参考。

**Q: 生成的内容太长/太短？**
A: 在大纲阶段就指定每部分的字数。如果已经生成，可以说"把这段压缩到 300 字"或"把这段扩充到 800 字"。

**Q: 想在中途改变方向？**
A: 没问题。说"停一下，我想调整大纲"，然后重新确认方向。

**Q: 如何继续编辑已生成的文章？**
A: 打开那篇文章，然后告诉 AI "继续写下一部分"或"重写第三段"。AI 会读取现有内容保持一致性。
